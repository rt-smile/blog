工具软件►git

git

文件

  • .gitignore
    记录要 git 忽略的文件
  • .gitkeep
    其实就起个占位作用。因为git不会跟踪、推送空文件夹。
    如果想要git追踪、推送某个空文件夹，则需要放一个文件在空文件夹中。
    其它名字也行。

git reset --hard [版本号]

回到指定版本，reset index and working tree

git reset (--mixed) [版本号]

回到指定版本，Resets the index but not the working tree

git reset

git add时，想要回退

git log ([分支名])

查看分支提交记录
如查看分支的远程仓库提交记录，origin/[仓库名]

git log -p FETCH_HEAD

查看 git fetch 取回的代码

git fetch 、 git pull

详情链接

git branch

查看存在的分支，并显示当前分支

git branch -a

查看所有分支，并显示当前分支

git branch [新分支名]

创建新分支

git switch [分支名]

切换分支

git push

提交commit到当前分支

git push origin HEAD:[分支名]

提交commit到仓库名为origin的指定分支

git push -f (origin [分支名])

强推当前commit
比如将回退的后版本强推到远程主机。此时如果用“git push”会报错，因为我们本地库HEAD指向的版本比远程库的要旧：

git commit -m “”

git add .

git add . ‘:!src/types/goods.d.ts’

添加除了某个路径下的文件

git config (--local) user.name ‘名字’

设置当前仓库git的用户名
write to the repository .git/config file. This is the default behavior.

git config --global user.name ‘名字’

设置git全局用户名
write to global ~/.gitconfig file rather than the repository .git/config

git config (--local) user.email you@example.com

git config --global user.email you@example.com

git commit --amend --reset-author

重设当前提交的用户和提交信息

git remote

显示已经添加的仓库名

git remote -v

显示已经添加的仓库名和对应的URL

git remote add [仓库名] [URL]

Add a remote named for the repository at
给远程仓库添加名字和对应的URL

git remote set-url [仓库名] [URL]

更换指定仓库的URL

git remote rename [旧远程名] [新远程名]

重命名 某个远程的名字

git remote remove [远程名]

移除某个远程
